Manuel Utilisateur

Avant de jouer

Ce jeu est la version originale d'un micro-projet fait dans le cadre d'un DUT informatique.

Il utilise une bibliothèque rudimentaire programmée par un professeur et ainsi ne marche que sous un Linux utilisant X-window. (Dans un futur proche, il est possible que Wayland remplace X-window).

J'ai essayé de rendre cette application portable, de sorte à ce qu'elle fonctionne sans installation.
Si vous rencontrez des problèmes, je vous invite à me contacter.

Indications

Lors de la modification de champs de texte dans le menu principal :

  • Echap permet d'annuler les modifications.
  • Entree ou Clic souris sert à valider.
  • Flèches gauche ou droite pour déplacer le curseur.
  • Backspace et Suppr. pour leur usage habituel.

La configuration

  • Ce fichier est à la fois un thème visuel de jeu et des paramètres de jeu, comme la taille de la grille, l'image de fond, le nombre de couleurs de billes, etc.
  • Différentes configurations sont prêtes à être testées, dans data/cfg/.
  • Vous pouvez créer la vôtre. Plus d'informations en bas.

La map

  • Ce fichier contient des informations sur la position des billes et leur couleur.
  • Si un fichier de map est introuvable (chemin vide inclus), alors la grille est générée aléatoirement.
  • Certaines maps sont prêtes à l'emploi dans data/map/.
  • Vous pouvez créer la vôtre. Plus d'informations en bas.

Le jeu

Le but est de gagner un maximum de points. Pour cela, il faut cliquer sur des groupes de billes de même couleur afin de les éliminer et former d'autres groupes.

Le nombre de points en fonction du nombre de billes est exponentiel ! La meilleure stratégie consiste à faire grossir un groupe d'une couleur puis l'éliminer une fois qu'il n'est plus possible de l'agrandir. Plus le paquet est gros, plus chaque bille apporte de points !

Consulter les scores

Un fichier de score est utilisé pour chaque configuration utilisée.

Il garde trace d'absolument tous les scores enregistrés pour la configuration et les trie par ordre décroissant.

Pour un fichier de configuration data/cfg/foo.cfg,

Son fichier de score est data/cfg/foo.cfg.scores.

C'est un fichier de texte, il est donc consultable. Une entrée est de la forme :

score <-- pseudo (map)

Créer une configuration

Le jeu vient avec quelques fichiers de configuration prêts à l'emploi. Ils se trouvent dans data/cfg/. Il est conseillé de copier une configuration existante, sous un autre nom, et de partir de là.
Cependant, l'ordre dans lequel les variables sont placées dans le fichier n'importe pas.
Voici la convention d'écriture d'un fichier .cfg pour ce jeu :

  • Tout ce qui suit un '#' est considéré comme un commentaire sur une ligne.
  • Lorsqu'une couleur est attendue, la noter en hexadécimal exclusivement.
    Ex. 0xFF0000
  • Pour palette_grille, une notation ensembliste est attendue. Entre accolades, les éléments sont des couleurs, séparés par des virgules. Le nombre d'éléments dans cet ensemble ne doit pas dépasser 6. La palette permet de remplacer une image de bille manquante (ou non souhaitée) par une jolie bille artisanale.
    Ex. {0xFF0000, 0x00FF00}
  • A chaque élément de cette palette, correspond un fichier image.
    Ex. 'image_bille_1' correspond à l'image de la première couleur de la palette.
  • Lorsqu'un chemin vers un fichier image est attendu, mettre le nom de fichier entre guillemets.
    Si aucune image n'est désirée, mettre les guillemets vides tout de même.
  • Pour le reste, un nombre entier non-signé est attendu.

Créer une map

Comme la configuration, c'est un bête fichier texte. Regardez les exemples de /data/map.

  • Les caractères 'R', 'V', 'B', 'C', 'J', 'M' vont remplir la bille avec respectivement les couleurs rouge, vert, bleu, cyan, jaune et magenta quelle que soit la palette de couleurs de la grille.
  • Les caractères '1', '2', '3', '4', '5', '6' vont remplir la bille avec la couleur de la palette de la grille, ayant pour index le chiffre.
  • les espaces génèrent un trou.
